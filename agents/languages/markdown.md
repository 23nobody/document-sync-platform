# Markdown Rules for Agents

## Scope

Applies to `docs/**/*.md`, `README.md`, and agent docs.

## Rules

1. Keep docs concise, scannable, and implementation-aligned.
2. Update docs in the same change when behavior or contracts change.
3. Use explicit field names for contracts and examples.
4. Avoid ambiguous terms; define assumptions and constraints clearly.
5. Do not include secrets, internal tokens, or sensitive sample payloads.

## PRD/Contract Updates

- If sync behavior changes, update operation semantics.
- If schema/canonical behavior changes, update examples and validation notes.
- Keep milestone and acceptance criteria internally consistent.
